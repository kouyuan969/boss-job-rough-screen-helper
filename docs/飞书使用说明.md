# BOSS 岗位粗筛助手使用说明

这是一份给使用者看的安装和使用说明。复制到飞书文档后，可以按需要再加截图。

## 这是什么

BOSS 岗位粗筛助手是一个 Chrome 扩展。

它用来在 BOSS 直聘岗位列表页做初步筛选：按岗位性质、目标方向、薪资、招聘者活跃度等条件判断岗位是否值得看。

它可以把比较匹配的岗位自动收藏，也会把判断结果记录下来，后面可以复制成表格继续复查。

## 它适合解决什么问题

适合这些场景：

- 每天岗位很多，不想一个个点开看
- 想先过滤掉明显不合适的岗位
- 想把可能合适的岗位先收藏起来
- 想保留筛选记录，后面再人工复盘
- 想用多关键词连续找岗位

它不是自动投递工具，也不是自动聊天工具。

## 它会做什么

插件会做：

- 读取当前 BOSS 岗位列表页上的岗位信息
- 根据你填写的岗位性质和目标方向打分
- 自动收藏达到分数线的岗位
- 跳过明确已经处理过的岗位
- 记录每个岗位的判断原因
- 导出表格、JSON 和诊断信息

插件不会做：

- 自动打招呼
- 自动投递
- 读取聊天内容
- 绕过验证码
- 绕过登录异常或账号安全提示

如果页面出现验证码、账号异常、频率提示，请停止使用。

## 安装方法

1. 打开 Chrome。
2. 在地址栏输入：

```text
chrome://extensions/
```

3. 打开右上角「开发者模式」。
4. 点击「加载已解压的扩展程序」。
5. 选择刚刚解压出来的插件文件夹。
6. 打开 BOSS 直聘岗位列表页。

BOSS 岗位列表页一般是这个格式：

```text
https://www.zhipin.com/web/geek/jobs
```

打开后，页面右侧会出现「BOSS 岗位粗筛助手」面板。

## 第一次怎么用

建议先小范围测试，不要一上来跑很多岗位。

可以这样设置：

1. 在 BOSS 页面顶部先选城市、薪资、公司规模等条件。
2. 在插件面板里填写岗位性质，例如：

```text
销售
```

3. 填写目标方向，例如：

```text
AI客服
```

或：

```text
AI应用
```

4. 检查正向词和反向词。
5. 把本次最多扫描数量先设小一点，比如 `10`。
6. 点击开始扫描。
7. 看最近判断和复制出来的表格，确认规则没有跑偏。

确认没问题后，再扩大扫描数量。

## 推荐填写方式

岗位性质不要写太泛。

推荐写：

- 销售
- BD
- 商务
- 大客户
- 客户经理
- 客户成功
- 售前

目标方向也不要只写 `AI`、`软件`、`互联网` 这种大词。

推荐写到具体场景：

- AI客服
- AI应用
- AI SaaS
- AI软件
- 智能客服
- 大模型应用
- Agent
- 企业软件

越具体，筛选结果越稳。

## 当前筛选规则

插件会重点看这些信息：

- 岗位标题和职位描述是否命中目标方向
- 岗位性质是否匹配
- 薪资是否在目标范围内
- 招聘者是否近期活跃
- 是否命中明确排除词

当前规则里：

- 招聘者超过 3 日未活跃，会排除。
- `在线`、`今日活跃`、`3日内活跃` 不会因为活跃度被排除。
- 日薪、时薪岗位会排除。
- 多段薪资会优先看和目标薪资有交集的一段。
- 已排除、已收藏、用户不想看的历史岗位会跳过。
- 待复核、收藏失败、用户想看的历史岗位，可以重新判断。

## DeepSeek 怎么用

不填 DeepSeek API Key 也可以用，插件会走本地规则。

如果想让 DeepSeek 辅助生成搜索词、加分词和排除词，可以在高级规则里填写自己的 API Key。

注意：

- 插件不会内置任何 API Key。
- API Key 由使用者自己填写。
- DeepSeek 只负责生成规则，不会直接替你投递或聊天。

## 导出的表格怎么看

点击「复制表格」后，可以粘到飞书表格或 Excel。

常用字段：

- `结果`：已收藏、待复核、已排除、收藏失败
- `推荐动作`：下一步建议
- `是否需要人工看`：是否需要你再看一眼
- `主要原因`：最主要的判断理由
- `分数`：岗位匹配分
- `命中关键词`：加分来源
- `扣分关键词`：排除或扣分来源
- `决策日志`：完整判断过程
- `链接`：岗位链接

建议优先看：

1. 已收藏
2. 待复核
3. 收藏失败

已排除的岗位一般不用看，除非你想检查规则是不是太严格。

## 使用建议

- 第一次使用先小批量测试。
- 不建议长时间无人值守运行。
- 不建议频繁启动停止。
- 看到验证码、账号异常、频率提示时，立即停。
- 规则不要一次写太多，先从少量正向词和排除词开始。
- 每次换方向后，先检查搜索词和排除词有没有冲突。

## 常见问题

### 1. 为什么有些岗位没有收藏？

可能原因：

- 分数没有达到收藏线
- 命中了排除词
- 薪资不匹配
- 招聘者太久没活跃
- 页面收藏按钮没有识别成功

可以看表格里的 `主要原因` 和 `决策日志`。

### 2. 为什么有些岗位进入待复核？

待复核通常表示岗位有一点相关，但不够确定。

比如：

- 标题不清楚，但描述里有目标方向
- 岗位性质接近，但不是完全匹配
- 薪资或业务场景需要人工判断

### 3. 为什么已经扫过的岗位还会再扫？

插件会直接跳过明确结果的岗位，比如已排除、已收藏、用户不想看。

但待复核、收藏失败、用户想看的岗位，可能会重新判断。因为规则更新后，它们有可能变成已收藏或已排除。

### 4. 这个插件安全吗？

它只在当前 BOSS 岗位列表页做本地筛选和页面操作。

但它仍然会点击收藏按钮，所以不要无限制运行。建议设置每日扫描和收藏上限。

### 5. 可以自动打招呼吗？

当前版本不做自动打招呼，也不做自动投递。

这样边界更清楚，风险也更低。

## 适合谁用

适合想系统筛岗位的人，尤其是：

- AI 应用销售
- AI 软件销售
- AI SaaS 销售
- AI 解决方案销售
- 客户成功、售前、BD 等相近方向

如果你的方向差异很大，也可以改岗位性质、目标方向、正向词和排除词后使用。

## 最后提醒

这个工具只做粗筛。

最终要不要投、怎么沟通、怎么判断公司，还是要人工确认。插件的价值是先帮你省掉一部分重复筛选时间。
